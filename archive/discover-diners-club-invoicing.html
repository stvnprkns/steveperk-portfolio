The 4 different user types that will use the platform in different capacities are:

Suppliers sell goods to the buyers.

The Work

Site Flow

From the start, we knew that this platform was going to be big‚Äîand there would be a lot of data to show. To help keep everything straight, I mapped out the points where the different user types crossed paths and what features each would use.

Modular Layout

Since requirements shifted a lot in the beginning of this project, we chose to build our application modularly. This helped us avoid creating one-off solutions. It also gave us the flexibility to adapt quickly and support many different types of content in multiple views, including tables, single-page components, and form fields.

Component Library

Diners Club only had marketing design guides‚Äîwhich meant there was no standard for web components. I used my experience creating component libraries to build them a new one from scratch.
